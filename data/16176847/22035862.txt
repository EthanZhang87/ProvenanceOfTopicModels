Apple Push Notification service APNs previously known as Apple Push Service APS is a platform notification service created by Apple Inc that enables third party application developers to send notification data to applications installed on Apple devices The notification information sent can include badges sounds newsstand updates or custom text alerts It was first launched with iOS on June APNs support for local applications was later added to the Mac OS X API beginning with the release of Mac OS X Lion Support for website notifications was later added with the release of Mac OS X Mavericks br br br History br Apple announced the service on June with an original stated release for that September however as stated by Scott Forstall at the iOS preview event on March the rollout was delayed after a decision to restructure the APNs for scalability purposes due to the allegedly overwhelming response to the announcement of the APNs At both events Forstall stated that push notifications better conserve battery than background processes which are used in pull technology for receiving notifications br APNs was first launched together with iOS on June The release of iOS included a Notification Center adding support for receiving and reading local notifications in a single place br APNs was also added as an API to Mac OS X Lion so that developers could begin updating their third party applications and start utilizing the service Support was later improved in OS X Mountain Lion with the introduction of a Notification Center As with iOS the improvement allowed users to manage and read their received notifications in a single location The release of OS X Mavericks included Safari which added support for accepting and receiving APNs notifications from websites that the user granted permission to br In December concerns arose regarding a potential privacy and surveillance loopholes involving push notifications delivered through APNs US Senator Ron Wyden revealed through a letter to the Department of Justice that both the US government and foreign law enforcement could demand user data from Apple related to push notifications br br br Technical details br In the maximum size allowed for a notification payload sent through the binary interface was increased from bytes to kilobytes In December a new HTTP provider API was released by Apple effectively replacing the now legacy binary interface The maximum notification payload size allowed using the HTTP API is kilobytes Apple shut down the legacy binary API at the end of March br The HTTP provider for APNs uses TCP port as the main port of communication but developers are also allowed to use TCP port if outbound access to port is blocked by firewalls br br br See also br iMessage br FaceTime br
