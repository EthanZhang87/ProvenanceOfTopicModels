Plack is a Perl web application programming framework inspired by Rack for Ruby and WSGI for Python and it is the project behind the PSGI specification used by other frameworks such as Catalyst and Dancer Plack allows for testing of Perl web applications without a live web server br Plackup is a command line utility to run PSGI applications from the command line br br br PSGI br PSGI or Perl Web Server Gateway Interface is an interface between web servers and web applications and frameworks written in the Perl programming language that allows writing portable applications that can be run as standalone servers or using CGI FastCGI mod perl et al It is inspired by the Web Server Gateway Interface for Python Rack for Ruby and JSGI for JavaScript A PSGI application is a Perl subroutine that accepts arguments as a single hash reference and returns a reference to an array of three elements an HTTP status code a reference to an array of HTTP headers and a reference to an array of HTTP body lines usually a generated HTML document or a filehandle like object br br br Supported backends br As of May Plack supports the following server backends br br CGI br SCGI br FastCGI br mod perl under Apache and br Standalone HTTP server included in Plack br HTTP Server Simple br Corona br Starman br Twiggy br AnyEvent HTTPD br AnyEvent ReverseHTTP br br br Examples br Using the default standalone HTTP server br br Running as a FastCGI daemon listening on a Unix socket ready to be used by any Web server with FastCGI support br br A working Hello world application run as a one liner br br The command above starts an HTTP server listening on port of every local interface IP address and returns this OK response to every HTTP request br br br br br br External links br PSGI and Plack website br Plack documentation br plackup manual
